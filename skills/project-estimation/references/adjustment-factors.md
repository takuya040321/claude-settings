# 調整係数リファレンス

## 1. 複雑度係数

### タスク複雑度

| 複雑度 | 係数 | 判断基準 |
|--------|------|----------|
| 低 | 0.7 | 標準的なCRUD、既存パターンの再利用、明確な仕様 |
| 中 | 1.0 | 一般的な機能、多少のカスタマイズ、標準的な連携 |
| 高 | 1.5 | 複雑なビジネスロジック、外部連携多数、パフォーマンス要件厳格 |

### 複雑度判定フロー

```
以下の項目をチェック:
□ 外部システム連携が3つ以上 → +0.2
□ リアルタイム処理が必要 → +0.2
□ 高度なセキュリティ要件 → +0.1
□ パフォーマンス要件が厳格 → +0.1
□ 複雑なビジネスルール → +0.2
□ 多言語・多通貨対応 → +0.1

合計が0の場合: 低
合計が0.1〜0.3の場合: 中
合計が0.4以上の場合: 高
```

## 4. 単価設定ガイドライン

### 単価算出方法（原価計算方式）

人件費ベースで単価を算出する方法。根拠を明確にしたい場合に使用。

#### Step 1: 月間人件費の算出

| 項目 | 計算方法 | 備考 |
|------|----------|------|
| 基本給（月給） | 市場相場を参照 | 下表参照 |
| 社会保険料（会社負担） | 月給 × 15% | 健康保険・厚生年金等 |
| 福利厚生・経費 | 月給 × 10% | 交通費・福利厚生等 |
| **月間人件費** | **上記合計** | |

**経験度別月給相場:**

| 経験度 | 月給目安 | 備考 |
|--------|----------|------|
| 初級（1〜2年） | 300,000〜350,000円 | 新卒〜若手 |
| 中級（3〜5年） | 400,000〜500,000円 | 一人称で開発可能 |
| 上級（5年以上） | 550,000〜700,000円 | リード経験あり |

#### Step 2: 時間原価の算出

```
月間稼働時間 = 稼働日数(20日) × 1日の稼働時間(8時間) = 160時間

時間原価 = 月間人件費 ÷ 月間稼働時間
```

**計算例（中級エンジニア）:**
```
月間人件費 = 450,000円 + 67,500円(15%) + 45,000円(10%) = 562,500円
時間原価   = 562,500円 ÷ 160時間 = 3,516円/時間
```

#### Step 3: 請求単価の算出

| 項目 | 割合 | 備考 |
|------|------|------|
| 時間原価 | 100% | Step 2で算出 |
| 利益率 | 20〜40% | 事業継続のための利益 |
| 管理コスト | 5〜15% | 営業・管理間接費 |

```
請求時間単価 = 時間原価 × (1 + 利益率 + 管理コスト)
請求人日単価 = 請求時間単価 × 8時間
```

**計算例（利益率30%、管理コスト10%）:**
```
請求時間単価 = 3,516円 × 1.4 = 4,922円 ≒ 5,000円
請求人日単価 = 5,000円 × 8時間 = 40,000円
```

#### 経験度別単価早見表

| 経験度 | 月給 | 時間原価 | 請求時間単価 | 請求人日単価 |
|--------|------|---------|-------------|-------------|
| 初級 | 350,000円 | 2,734円 | 3,800円 | 30,400円 |
| 中級 | 450,000円 | 3,516円 | 5,000円 | 40,000円 |
| 上級 | 600,000円 | 4,688円 | 6,600円 | 52,800円 |

※利益率30%、管理コスト10%で計算

### 人日単価の算出

```
人日単価 = 人月単価 / 20営業日

例: 90万円/月 ÷ 20日 = 4.5万円/日
```

### 見積もり金額の算出

```
見積もり金額 = 工数(人日) × 人日単価

例: 30人日 × 4.5万円 = 135万円
```

## 5. 期間算出

### 工数から期間への変換

```
期間(月) = 工数(人日) / (チーム人数 × 20営業日 × 稼働率)
```

**稼働率の目安:**
- 専任: 80〜90%
- 兼任: 50〜70%
- パートタイム: 30〜50%

### 計算例

```
工数: 40人日
チーム: 2人（専任80%）

期間 = 40 / (2 × 20 × 0.8)
     = 40 / 32
     = 1.25ヶ月
     ≒ 約1.5ヶ月（バッファ含む）
```

### クリティカルパスの考慮

並行作業が困難なタスクがある場合、期間が延びる可能性あり:
- 要件定義 → 設計 → 実装（直列）
- フロントエンド・バックエンド同時開発（並列可能）
- テスト → 修正 → 再テスト（直列）

## 6. 総合計算例

### プロジェクト: ECサイト構築

**前提:**
- チーム: 2名（中級）
- 技術: Next.js + Node.js + PostgreSQL + Stripe
- ドメイン知識: 少しあり

**積み上げ法:**
```
基準工数: 30人日
複雑度調整: 30 × 1.2 = 36人日
技術係数: (1.0 + 1.0 + 1.0 + 0.9) / 4 = 0.975
チーム係数: 1.0 × 0.95 × 1.1 = 1.045

最終工数: 36 × 0.975 × 1.045 = 36.7人日
バッファ込み: 36.7 × 1.2 = 44人日
```

**費用算出:**
```
人日単価: 4.5万円（ミドル標準）
開発費: 44人日 × 4.5万円 = 198万円
PM費(10%): 19.8万円
---
合計: 約220万円
```

**期間:**
```
期間: 44 / (2 × 20 × 0.8) = 1.375ヶ月
→ 約1.5〜2ヶ月
```
