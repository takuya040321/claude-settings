# プロジェクト見積もりスキル

## 概要

ソフトウェア開発プロジェクトの見積もりを、複数の手法を用いて算出し、比較可能な形式で出力します。

**対応する見積もり手法:**
- 積み上げ法（ボトムアップ）
- 三点見積もり（PERT法）
- 類推法（過去プロジェクト参照時）

## ワークフロー

```
[要件収集] → [機能分解] → [見積もり算出] → [出力]
     ↓            ↓             ↓           ↓
  ヒアリング    WBS作成     複数手法で算出   MD形式
```

### Phase 1: 要件収集

ユーザーから以下の情報を収集する：

| 必須項目 | 説明 | 例 |
|---------|------|-----|
| プロジェクト名 | 見積もり対象の名称 | 「ECサイト構築」 |
| 機能要件 | 実装すべき機能の一覧 | ユーザー認証、商品検索、決済 |
| 技術スタック | 使用する技術 | React, Node.js, PostgreSQL |

| 任意項目 | 説明 | デフォルト |
|---------|------|-----------|
| チーム規模 | 開発者数 | 1人 |
| チーム経験度 | 該当技術の経験 | 中級 |
| 納期制約 | 希望納期 | なし |
| 予算制約 | 上限予算 | なし |
| 類似プロジェクト | 過去の参考案件 | なし |

**収集方法:**
1. ユーザーの最初のメッセージから抽出可能な情報を特定
2. 不足している必須項目があれば質問
3. 任意項目は提供された場合のみ使用

### Phase 2: 機能分解（WBS作成）

収集した要件を作業分解構造（WBS）に展開する。

**分解の粒度:**
- レベル1: 大機能（例: ユーザー管理）
- レベル2: 中機能（例: 認証、プロフィール）
- レベル3: 小機能（例: ログイン、ログアウト、パスワードリセット）

**各タスクに設定する属性:**
- タスク名
- 複雑度（低/中/高）
- 基準工数（人日）

詳細は `references/estimation-methods.md` を参照。

### Phase 3: 見積もり算出

3つの手法で見積もりを算出する。

#### 手法1: 積み上げ法

各タスクの工数を合計し、調整係数を適用。

```
総工数 = Σ(タスク基準工数 × 複雑度係数) × 技術係数 × チーム係数
```

#### 手法2: 三点見積もり（PERT法）

各タスクに楽観値・最頻値・悲観値を設定し、期待値を算出。

```
期待値 = (楽観値 + 4×最頻値 + 悲観値) / 6
標準偏差 = (悲観値 - 楽観値) / 6
```

#### 手法3: 類推法

類似プロジェクトの実績から類推（情報が提供された場合のみ）。

詳細な計算方法は `references/estimation-methods.md` を参照。
調整係数の一覧は `references/adjustment-factors.md` を参照。

### Phase 4: 出力
